# 6. 用語集: モデル・プロンプト・コンテキストほか

生成AIの記事を読んでいると、同じ会話を指して「Claudeが答えた」「Anthropicが答えた」「このモデルが答えた」と、呼び方がコロコロ変わることに気づきます。読者の皆さんが悪いのではなく、業界の語彙自体がまだ揺れているためです。本章では、以降の章でつまずかないために、最低限おさえておきたい6つの言葉を整理します。辞書として戻ってきてもらえれば事足りるので、すべて暗記する必要はありません。

## 対象読者と前提

- 生成AIの記事や社内資料で用語に詰まったことがある人
- 他章を読む途中で「この単語、どういう意味だったか」と立ち止まった人

本章は**辞書のように使える構成**にしてあります。章の流れに沿って通読しても構いませんし、目次から必要な項目だけ拾い読みしても困らないようにしてあります。1〜5章を読んでから開いた場合は、すでに触れている概念に正しいラベルを貼り直す作業になります。本章から先に開いたなら、用語を眺めたうえで他章へ進むと、以降の読み味が軽くなります。

## モデルとプロバイダ

この2つは、セットで覚えると混乱しません。

- **モデル** — 文章を実際に生成する「頭脳」そのもの。バージョンが上がると別人格と言えるほど振る舞いが変わる
- **プロバイダ** — そのモデルを作り、サービスとして提供している会社や組織

たとえば「Claude」はAnthropic社が提供する**モデルのブランド名**であり、その中に `Claude Opus` `Claude Sonnet` `Claude Haiku` のように**具体的なモデル**が並んでいます。「Gemini」も同じで、Google社がプロバイダ、その下に `Gemini Pro` や `Gemini Flash` が並んでいます。

| 区分 | 例（2026年4月時点の代表例） |
| ---- | ---- |
| プロバイダ | Anthropic、Google、OpenAI |
| モデルのブランド | Claude、Gemini、GPT |
| 具体的なモデル | Claude Opus、Gemini Pro、GPT-5 |

ブランド名と具体的なモデル名の使い分けは、車で言えば「トヨタのプリウス」と「プリウス」くらいの関係だと思っておけば通じます。普段の会話ではブランド名だけで足り、性能や料金の話になると車種名まで指定する必要が出てきます。

## プロンプト

**プロンプト**は、AIに対して渡す指示文のことです。チャット欄にあなたが打ち込む文章そのもの、と思ってもらって構いません。

ポイントは3つあります。

- プロンプトは「命令文」ではなく「依頼文」。前提を丁寧に書くと品質も上がる
- 役割、背景、出力形式を分けて書くと、AIが指示を拾いやすくなる
- 長ければよいわけではなく、**必要な情報だけ**を過不足なく渡すのが大事

「よいプロンプトの書き方」は7章で詳しく扱います。本章では「プロンプト＝AIへの依頼文」とだけ押さえていただければ、先の章に進んで差し支えありません。

## コンテキスト

**コンテキスト**は、AIがいまの回答を作るために参照している情報の総体です。もう少し具体的に言うと、以下のすべてがコンテキストに入ります。

- あなたが今回打ち込んだプロンプト
- その会話のこれまでのやり取り（直前のQ&Aもふくめて）
- システムプロンプトと呼ばれる、サービス側があらかじめ仕込んだ指示
- アップロードしたファイルや、コネクタ経由で差し込まれた資料

ここで、コンテキストには上限があります。この上限のことを**コンテキストウィンドウ**と呼び、モデルごとに決まっています。大きすぎる資料を丸ごと投げ込むと、この窓からはみ出した部分は**単純に無視されます**。

読書に例えるなら、コンテキストは机の上に広げられた資料の山です。机に乗り切らない分量は読み込みようがありません。AIにとっての「机の広さ」がコンテキストウィンドウにあたります。

## トークン

**トークン**は、AIが文章を扱うときの最小単位です。英単語ひとつが1トークン、日本語は文字種によって2〜3文字で1トークン、といった粒度だと理解しておけば、普段の利用には十分です。

トークンという言葉が頻繁に出てくる理由は、主に2つあります。

- **課金の単位** — APIから利用する場合、入力トークン数と出力トークン数にもとづいて料金が決まる
- **上限の単位** — コンテキストウィンドウも「何トークンまで入るか」で表される

普段の業務利用では、トークンの実数を気にする場面はさほど多くありません。ただし「資料を全文投げたら途中で切れた」「APIの請求がなぜか高い」といった事態に出会ったとき、トークンという物差しを知っておくと、原因の切り分けが進めやすくなります。

## エージェント

**エージェント**は、AIが単にチャットで答えるのではなく、**目的を達成するために自分で手を動かすモード**のことを指します。手を動かすとは比喩ではありません。実際にツールを呼び、ファイルを読み書きし、場合によってはブラウザまで操作する、という意味です。

チャットとの違いを単純化すると、以下のようになります。

| 観点 | 従来のチャット | エージェント |
| ---- | ---- | ---- |
| ユーザーの関与 | 一問一答のたびに人が判断 | 途中の判断はAIが担う |
| 外部操作 | 原則なし（または確認つき） | AIが能動的にツールを呼ぶ |
| 成果物 | 会話の返答 | タスクの完遂（ファイル、投稿、予約など） |

エージェントは便利ですが、同時に「AIが知らないところで操作を走らせる」ため、セキュリティ上の注意点が一段階増えます。この点は9章で集中的に扱います。

## 用語一覧（戻ってきたいとき用）

章を読み進める途中で「あれ、この単語なんだったか」となったら、この表に戻ってきてください。

| 用語 | 一言で | 次に出てくる場所 |
| ---- | ---- | ---- |
| モデル | 文章を生成する本体 | 10章、12章（個別モデルの使いこなし） |
| プロバイダ | モデルを提供する会社 | 8章（データの扱い） |
| プロンプト | AIへの依頼文 | 7章（書き方のコツ） |
| コンテキスト | AIがいま参照している情報の総体 | 7章、9章 |
| トークン | 文章の最小単位、課金と上限の単位 | 10章、12章（料金と制限） |
| エージェント | AIが自分で手を動かすモード | 3章（仕組み）、9章（ガバナンス） |

## まとめ

- モデルとプロバイダ、プロンプトとコンテキストのように、**セットで意味が決まる用語**はペアで覚えると混乱が減る
- トークンを普段から意識する必要はないが、**料金と上限の話になると一気に主役**になる
- エージェントは便利さと引き換えに責任範囲が広がる。セキュリティ章（8章・9章）までセットで読んでほしい
