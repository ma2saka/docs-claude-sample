# Appendix: Claude Code

Claude Codeは、Anthropic社のClaude製品の1つで、利用者のPC上のフォルダや資料の整理、定型作業の自動化までを会話から頼めるよう設計されています。本付録は、ブラウザ版のClaudeから一歩進んでClaude Codeを利用するときの全体像、チャット版との違い、最初の利用方針を扱います。

開発寄りに作られた道具ですが、非エンジニアの仕事にも当てはまる利用範囲があります。本付録の後半は、その範囲を中心に扱います。

## 対象読者と前提

- [13章](13-claude.md)でブラウザ版Claudeを一通り利用した人
- [Appendix: ワークフローツール](appendix-workflow-tools.md)で、生成AIが処理の1ステップとして動く構成に目を通した人
- 「コードは書かないが、PC上のファイルや定型作業までAIに任せてみたい」立場の人

コマンドラインを日常的に利用する必要はありません。必要に応じてターミナル（コマンドを打ち込む画面）を開く場面もある、くらいの前提でお読みください。

Claudeの4製品の役割分担は、[13章「Claudeの主な4製品はブラウザ版とPC上で動く3つ」](13-claude.md)の表と図に整理されています。本付録は、そのうちClaude Codeを掘り下げる構成です。13章の該当節の表と図は、本付録で踏み込む話のスケール感を見定める基準として参照できます。

## Claude Codeには4つの提供形態があり、中身は共通している

**Claude Code**は、Anthropic社が提供する公式のClaude製品の1つです。CLIから出発した道具で、現在はデスクトップアプリ・Webアプリ・IDE拡張も同じClaude Codeブランドで提供されています。会話と資料作成を主眼に置くブラウザ版Claudeに対して、Claude Codeはフォルダ単位のファイル操作やコマンド実行までを担う方向で設計されています。

提供形態は次の4つで、利用感と動作する場所が形態ごとに異なります。

| 提供形態 | 利用感の概要 |
| ---- | ---- |
| CLI（ターミナル） | 最初に提供された形態。手元のファイルとコマンドを直接対象にする |
| デスクトップアプリ（Mac／Windows） | GUIからClaude Codeの機能を呼び出す |
| Webアプリ（`claude.ai/code`） | ブラウザからクラウド上のリポジトリを対象に操作する。手元のファイルは扱わない |
| IDE拡張（VS Code／JetBrains） | エディタの中でそのまま呼び出す |

CLI・デスクトップアプリ・IDE拡張の3つは利用者のPC上で動作し、許可した範囲のローカルファイルを読み書きします。Webアプリ（`claude.ai/code`）はクラウド上の隔離されたコンテナで動作し、GitHubリポジトリをクローンしたうえでファイル操作やコマンド実行を担います。手元のPCのファイルは読み書きしませんが、リポジトリ単位の作業はPC版とほぼ同等に進められます。中身（モデルと振る舞いの骨格）はどの形態でも共通しています。

非エンジニアの利用者は、まずデスクトップアプリかWebアプリから始めると、ターミナル操作やインストール作業の負担なくClaude Codeの動きを把握できます。CLI版は、ターミナルでの操作に抵抗がなく、キーボード主体で作業を進めたい人に向きます。

## チャット版との違いは実行場所と権限範囲にある

本節以降では、PC上で動くClaude Code（CLI・デスクトップアプリ・IDE拡張）を念頭に置いて、ブラウザ版Claudeとの差を整理します。Webアプリ（`claude.ai/code`）は手元のPCには直接アクセスしませんが、クラウド上のリポジトリに対してはファイル操作とコマンド実行が可能です。Webアプリの扱いは本節の末尾で補足します。

両者の差は、回答の品質ではなく、動作環境と利用者のPCに対する権限の範囲です。対比は次のとおりです。

| 観点 | ブラウザ版Claude | Claude Code（PC側） |
| ---- | ---- | ---- |
| 実行場所 | Anthropic側のサーバ | 利用者のPC（CLI・デスクトップアプリ・IDE拡張） |
| ファイルの扱い | アップロードしたものだけ | 許可した範囲のローカルファイルを読み書き |
| コマンド実行 | 原則なし | 許可した範囲でシェルコマンドを実行 |
| 向く作業 | 会話、資料作成、1ファイルの編集 | フォルダ全体の変更、スクリプト作成、連続作業 |

呼び出すモデルは同じで、Claude Codeはこれに加えてローカルのファイル読み書きとコマンド実行を担います。利用者のPC上で一連の作業を連続して進めたい場面が、Claude Codeを選ぶ判断軸になります。

### Webアプリはクラウド上でリポジトリを操作する

Webアプリ（`claude.ai/code`）は、上の表のPC版に近い能力をクラウド上で提供します。GitHubリポジトリをクラウド上の一時的なコンテナにクローンし、そのコンテナの中でファイルの読み書きやコマンド実行を担う仕組みです。作業が終わったら、変更をコミットしてGitHubにプッシュします。

PC版との主な違いは、操作対象が手元のファイルではなくGitHubリポジトリである点です。ブラウザだけで始められるため、手元にインストール環境がない場面や、外出先からリポジトリの作業を進めたい場面に向きます。コンテナはセッション終了後に破棄されるため、プッシュしていない変更は失われます。

一方、手元のPCにあるファイル（ダウンロードフォルダの資料やローカルのCSVなど）を直接操作したい場合は、PC版を使います。Webアプリの操作対象はGitHubリポジトリに限られるため、手元のファイルの整理にはPC版が適しています。

## 最初の試用は空フォルダで会話を始める

PC版の場合、もっとも手軽な始め方は、公式ガイドに沿ってデスクトップアプリかCLIを入れ、空のフォルダで会話を開始することです。デスクトップアプリは公式サイトのインストーラを使うため、ターミナル操作は不要です。インストール手順は公式ドキュメントを一次ソースにしてください。CLIを選ぶ場合の最短ルートは、Node.jsが入っているPCで次の1行です。

```bash
npm install -g @anthropic-ai/claude-code
```

起動後、フォルダを指定してClaudeに話しかけると、そのフォルダの中のファイルをClaude Codeが参照します。たとえば、議事録ファイルを並べたフォルダで次のように依頼すると、要約の下書きが返ってきます。

```text
このフォルダの議事録を読んで、先月の意思決定を箇条書きで要約してください。
出典としてファイル名も併記してください。
```

ここで確認しておきたいのは、Claude Codeが実際にどのファイルを参照し、どのコマンドを実行しようとしているかが画面上に逐一表示される点です。利用者は表示された内容に対して許可／拒否を選びます。Claude Code側が利用者の同意を経ずに操作を進める作りではありません。

Webアプリ（`claude.ai/code`）を使う場合は、インストール不要です。ブラウザで`claude.ai/code`を開き、対象のGitHubリポジトリを選んでセッションを開始します。リポジトリがクラウド上のコンテナにクローンされ、以降はPC版と同じ感覚でファイル操作やコマンド実行を依頼できます。

## Claude Codeに固有の4用語は名前と役割の対応で足りる

Claude Codeを少し深く利用したくなった場面で目にする用語のうち、最初に確認しておきたいものが4つあります。深く理解する必要はなく、名前と役割が頭にあれば、ドキュメントや画面の指示で迷いにくくなります。

| 用語 | 一言で | 初見でのおすすめ度 |
| ---- | ---- | ---- |
| スラッシュコマンド | `/help` のように先頭に `/` を付けて呼ぶ定型処理 | 高（まず `/help` を打つ） |
| サブエージェント | 役割を限定したClaude Codeを呼び出す仕組み | 中（慣れてから） |
| MCPサーバ | 外部サービスとClaude Codeをつなぐ共通の差込口 | 中（連携先ができてから） |
| フック | 特定のタイミングで自動的に動く処理 | 低（手元の操作になじむまでは利用しない） |

### スラッシュコマンド

画面内で `/` を入力すると、利用できるコマンドの一覧が表示されます。最初に確認しておくのは `/help` です。迷ったときは `/help` に戻ると、利用できるコマンドの一覧を再表示できます。

### サブエージェント

ひとつの会話の中で「コードを読む担当」「テストを実行する担当」のように、役割を限定したClaude Codeを呼び出せる仕組みです。[7章](07-terminology.md)で扱ったエージェントの考え方を、Claude Code側で構成した形です。

### MCPサーバ

MCPはModel Context Protocolの略で、[4章](04-external-system-integration.md)で扱ったツール呼び出しを共通化したプロトコルです。連携先ごとにMCPサーバを用意しておくと、Claude Codeから同じ作法でアクセスできます。

### フック

「コマンドを実行する直前にリントを実行する」「会話が終わったら通知を出す」のように、特定のタイミングで自動的に動く処理を差し込む仕組みです。便利ですが、設定ミスが意図しない動作につながりやすいため、利用は手元の操作になじんでからにします。

## 非エンジニアの実用は資料整理と定型作業の自動化に集中する

「自分は開発者ではない」という立場の利用者にとって、Claude Codeの利用価値は、コードを書くこと自体よりも、PC上の定型作業をまとめて頼める点に現れます。具体例として、次のような仕事が射程に入ります。

- ひとつのフォルダに集めた資料（議事録、ヒアリングメモ、CSV）の要約と横断比較
- スクリーンショットや写真のリネームと、日付別のフォルダ分け
- 調査の下書き（章立て付きのMarkdownで出力し、利用者が清書する進め方）
- 簡単なスクリプトの生成と動作確認（「このCSVから重複を除いたリストを出して」など）
- Google WorkspaceやSlackへの連携（MCP経由でコネクタを足してから）

一方、Claude Codeに向かない作業もあります。デザインの最終仕上げ、長尺の動画編集、文体の細かな推敲などは、ブラウザ版Claudeや専用ツールのほうが順当です。APIや連携機能が公開されていないアプリの画面越しの操作や、ランチャーやショートカットに生成AIを1ステップだけ差し込む構成は、別アプローチとして[Appendix: デスクトップの自動化](appendix-desktop-automation.md)で扱います。

## 最初の利用方針は「小さく分けて、人の確認を経由する」

CLIやMCPといった開発者向けの用語が並びますが、Claude Codeは利用者のPCに対してファイルの読み書きとコマンドの実行を担う道具です。次の点を最初に前提として共有しておくと、意図しない操作の影響範囲を狭く保てます。

- 許可ダイアログを読み飛ばさない — ファイルの書き込みやコマンド実行の許可ダイアログには、目を通してから承認する
- 対象フォルダを分ける — 本番のデータが入った場所では試さず、一時的な作業用フォルダを用意する
- Gitなどの履歴の下に置く — 生成結果は履歴を残し、あとから差し戻せる状態を保つ
- 個人情報・顧客情報の扱いは別途確認する — [9章](09-security-individual.md)と[10章](10-security-agent-era.md)に目を通したうえで、機微なデータに利用する
- 自動化は段階的に広げる — 最初からフックや自走エージェントを組むと、設定ミスから意図しない動作が起きた場合の影響範囲が大きくなる

以上を外さないうちは、Claude Codeの利用範囲は意図したものから外れにくく保てます。

## ブラウザ版・IDE拡張との使い分けは作業の規模で決める

同じClaudeブランドの中でも、動作環境と扱える範囲が違うため、向く作業も変わります。仕事に応じた使い分けの目安は次のとおりです。

| 状況 | 第一選択 | 補足 |
| ---- | ---- | ---- |
| 会話しながら1ファイルを仕上げたい | ブラウザ版Claude | Artifactsで成果物を整理する |
| 自分のPCのファイルをまとめて操作したい | Claude Codeデスクトップ／CLI | フォルダ単位で許可する |
| コードをエディタ内で編集したい | IDE拡張（VS Code／JetBrains） | エディタ側の差分表示をそのまま利用できる |
| GitHubリポジトリの作業をブラウザだけで進めたい | Webアプリ（`claude.ai/code`） | PCへのインストール不要。クラウド上のコンテナで操作する |
| 外出先で会話と資料作成に集中したい | ブラウザ版Claude | ファイル操作は対象外。Artifactsで成果物を整理する |

選択の軸は「どれが優れているか」ではなく、「いま向き合っている作業の規模に合うか」です。仕上げたいファイルが1つなのか、フォルダ横断の連続作業なのかで、適した呼び出し方は変わります。Claudeブランドの製品には、このほかにAPIや連携機能が公開されていないアプリの画面操作を担うClaude Computer Useもあります。Claude Computer Useがどの型に位置するかは、[Appendix: デスクトップの自動化](appendix-desktop-automation.md)の3アプローチ（ランチャー・ショートカット型／画面操作エージェント型／スクリプト生成・実行型）の中で扱います。

## まとめ

- Claude CodeはAnthropic公式のClaude製品の1つで、PC側ではローカルファイルの読み書きとコマンド実行を、Webアプリではクラウド上のGitHubリポジトリに対する同等の操作を担う
- 提供形態はCLI・デスクトップアプリ・Webアプリ・IDE拡張の4つで、中身は同じ
- 最初は小さなフォルダで試し、許可ダイアログを目視で確認するだけで、意図しない操作の多くは避けられる
- 非エンジニアでも、資料整理・スクリプト生成・SaaS連携に実用的な範囲がある
