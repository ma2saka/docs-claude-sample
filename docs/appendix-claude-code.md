# Appendix: Claude Code

ブラウザでClaudeと会話するのにようやく慣れた頃、どこかで「Claude Code」という単語が飛び込んできて、気になっている読者もいるはずです。本付録は、その正体と使い始めの入口を、**非エンジニアの視点からできるだけ遠回りせずに**案内するためのものです。

Claude Codeは、名前のとおりソフトウェア開発を主戦場に作られた道具ですが、視点を少しずらすと「自分のPCの中で動くClaude」として、エンジニア以外にも便利な場面があります。そのバランス感を伝えるのが、この付録の役目です。

## 対象読者と前提

- 12章でチャットとしてのClaudeを一通り触った人
- Appendix「ワークフローツール」で、生成AIが1ノードとして働く絵を頭に入れた人
- 「コードは書かないが、PC上のファイルや台本作りまでAIに任せてみたい」人

コマンドラインを日常的に叩く必要はありません。たまに真っ黒な画面を開く覚悟が要る、くらいの温度感でお読みください。

Claude Codeは、ブラウザ版Claude／Cowork／Computer Useと並ぶ**PC寄りの3兄弟**の1つです。4つの顔ぶれを横並びで把握したい場合は、[12章の「Claudeファミリー俯瞰」節](12-claude.md)に表と階段図を置いてあるので、先に眺めておくと位置関係を掴みやすくなります。

## Claude Codeとは何者か

Claude Codeは、Anthropic社が提供する**Claude公式のコマンドラインツール**です。ブラウザ版のClaudeが「向こうのサーバにある画面」だとすれば、Claude Codeは「自分のPCの中で動き、手元のファイルを触れるClaude」です。

一方でClaude Codeは、コマンドラインだけの存在ではありません。現在はいくつかの入口が用意されています。

| 入口 | ざっくりの使い心地 |
| ---- | ---- |
| CLI（ターミナル） | 本家。手元のファイルとコマンドに直接触れる |
| デスクトップアプリ（Mac／Windows） | GUIからClaude Codeの機能を呼び出す |
| Webアプリ | ブラウザ内でほぼ同等の体験。ローカルのファイルは扱わない |
| IDE拡張（VS Code／JetBrains） | エディタの中でそのまま呼び出す |

中身はどれも同じClaude Codeで、入口が違うだけです。非エンジニアの方は、まずデスクトップアプリか、エディタ拡張のどちらかから始めるのが穏当でしょう。「黒い画面から逃げない」と決めた人にだけ、CLI版をおすすめします。

## チャット版との違い

ブラウザ版のClaudeとの違いを、表で押さえておきます。

| 観点 | ブラウザ版Claude | Claude Code |
| ---- | ---- | ---- |
| 動く場所 | Anthropic側のサーバ | 自分のPC（またはIDE／デスクトップアプリ） |
| ファイルの扱い | アップロードしたものだけ | 許可した範囲の**ローカルファイル**を読み書き |
| コマンド実行 | 原則なし | 許可した範囲でシェルコマンドを実行できる |
| 得意な仕事 | 会話、資料作成、1ファイルの編集 | リポジトリ全体の変更、スクリプト作成、連続作業 |

簡単に言えば、**手元のPC上で一連の作業をまとめて任せたい**ときに強みが出るのはClaude Codeです。会話の頭脳は同じままで、腕と足がついている、というイメージで捉えておくとズレにくくなります。

## 最初の15分で試せること

いちばん手軽なのは、公式ガイドに沿ってCLIかデスクトップアプリを入れ、空のフォルダで会話を始めてみることです。インストール手順は公式ドキュメントを一次ソースにしてください。執筆時点での最短ルートは、Node.jsが入っているPCで次の1行です。

```bash
npm install -g @anthropic-ai/claude-code
```

起動後、フォルダを指定してClaudeに話しかければ、そのフォルダの中のファイルをClaude Codeが読みにいきます。たとえば、議事録ファイルを並べたフォルダで次のように頼むと、それらしい下書きを返してくれます。

```text
このフォルダの議事録を読んで、先月の意思決定を箇条書きで要約してください。
出典としてファイル名も併記してください。
```

この段階で見ておきたいのは、**Claude Codeが実際にどのファイルを読み、どのコマンドを動かそうとしているか**を逐一確認できる点です。画面の指示に従って許可／拒否するだけで、勝手に操作を進めるタイプの道具ではありません。

## 覚えておきたい4つの概念

Claude Codeを少し深く使いたくなったときのために、**名前だけでも頭に置いておくと用語に戸惑わない**4つの言葉を紹介します。辞書として戻ってきてもらえれば十分です。

| 用語 | 一言で | 初見でのおすすめ度 |
| ---- | ---- | ---- |
| スラッシュコマンド | `/help` のように先頭に `/` を付けて呼ぶ定型処理 | 高（まず `/help` を打つ） |
| サブエージェント | 特定の役割に特化した子分のClaude | 中（慣れてから） |
| MCPサーバ | 外部サービスとClaude Codeをつなぐ共通の差込口 | 中（連携先ができてから） |
| フック | 特定のタイミングで自動的に走る仕掛け | 低（手になじむまでは必要ない） |

### スラッシュコマンド

画面内で `/` を入力すると、使えるコマンドの一覧が出ます。最初に打っておきたいのは `/help` です。迷ったら `/help` に戻る、くらいの距離感で使えば差し支えありません。

### サブエージェント

ひとつの会話の中で、「コードを読む担当」「テストを走らせる担当」のように、役割を分けたClaudeを呼び出せる仕組みです。6章で触れたエージェントの考え方を、Claude Code側で形にしたものだと思ってください。

### MCPサーバ

MCPはModel Context Protocolの略です。3章で出てきた「ツール呼び出し」を、**プロトコルとして共通化したもの**と捉えるのが分かりやすい切り口です。連携先ごとにMCPサーバを用意しておくと、Claude Codeから同じ作法でアクセスできます。

### フック

「コマンドを実行する直前にリントを走らせる」「会話が終わったら通知を出す」など、**特定のタイミングで自動的に動く処理**を差し込む仕組みです。便利ですが、設定ミスが事故に直結しがちなので、最初のうちは手を出さなくて構いません。

## 非エンジニアの使いどころ

「自分は開発者ではない」読者に対して、Claude Codeの最大の魅力は**コードそのものではなく、PC上の定型仕事をまとめて頼める**点にあります。具体例を挙げると、次のような仕事が射程に入ります。

- ひとつのフォルダに集めた資料（議事録、ヒアリングメモ、CSV）の要約と横断比較
- スクリーンショットや写真のリネームと、日付別のフォルダ分け
- 調査の下書き（章立て付きのMarkdownで出力し、自分で清書する進め方）
- 簡単なスクリプトの生成と動作確認（「このCSVから重複を除いたリストを出して」など）
- Google WorkspaceやSlackへの連携（MCP経由でコネクタを足してから）

一方で、Claude Codeが得意でない領域もあります。「デザインの最終仕上げ」「長大な動画の編集」「手触りが命のライティング」などは、ブラウザ版Claudeや専用ツールのほうがスムーズに進みます。道具は適材適所です。

## 始めるときの注意点

CLIやMCPといった語感に気圧されて忘れがちですが、Claude Codeは**自分のPCに手を入れる道具**です。次の点だけは、最初に意識合わせしておくと事故が減ります。

- **許可ダイアログを読み飛ばさない** — ファイルの書き込みやコマンド実行の許可ダイアログは、目を通してから承認する
- **リポジトリやフォルダを分ける** — 本番のデータが入った場所で「試してみる」のは避け、一時的な作業用フォルダを用意する
- **Gitなどの履歴の下に置く** — 生成結果は履歴を残す。あとから差し戻せる状態を保っていると、踏み込んだ指示もしやすくなる
- **個人情報・顧客情報の扱いに注意** — 8章「個人利用編」と9章「エージェント時代のガバナンス」の整理にあらかじめ目を通してから、機微なデータに使う
- **自動化は小さく始める** — 最初からフックや自走エージェントを組むと、意図しない操作の影響範囲が大きくなる

ルールが多く見えますが、**要約すると「小さなフォルダで、人の目を通しながら、徐々に広げる」**の1行です。この1行を外さないうちは、Claude Codeとの付き合いは素直な手触りのまま進みます。

## ブラウザ版やIDE拡張との使い分け

同じClaudeブランドの中でも、入口の性格が違うため、得意分野も変わります。仕事に応じた使い分けの目安は次のとおりです。

| 状況 | 第一選択 | 補足 |
| ---- | ---- | ---- |
| 会話しながら1ファイルを仕上げたい | ブラウザ版Claude | Artifactsで成果物を整理する |
| 自分のPCのファイルをまとめて触りたい | Claude Codeデスクトップ／CLI | フォルダ単位で許可する |
| コードをエディタ内で編集したい | IDE拡張（VS Code／JetBrains） | エディタ側の差分表示が活きる |
| 外出先でPCが手元にない | ブラウザ版Claude／Webアプリ | ファイル操作は諦めて資料生成に集中 |

「どれが一番すごい」ではなく、「いま手にしている道具のサイズ感がどれくらいか」で選んでください。家具を組み立てるのに電動ドリルを出すか、プラスドライバーで済ませるか、くらいの話です。

## まとめ

- Claude Codeは**自分のPCに手を入れるClaude**であり、ブラウザ版との差は「腕と足の有無」
- CLI・デスクトップアプリ・Webアプリ・IDE拡張の4つの入口があり、中身は同じ
- 最初はフォルダ単位で小さく試し、**許可ダイアログを読み飛ばさない**だけで事故の多くは避けられる
- 非エンジニアでも、資料整理・スクリプト生成・連携作業で実用的な使い方が広がる
- フックやMCPは便利だが、**普段使いの土台ができてから**足すのが無難
