# Appendix: Claude Code

本付録は、Claude製品群のうち自分のPC上で動く「Claude Code」に焦点を当て、非エンジニアの読者が使い始めるときの最短経路と注意点を整理します。Claude Codeは元来ソフトウェア開発を想定した道具ですが、ローカルのファイル整理や簡単なスクリプト作成を任せたい場面では、開発者以外にも使い道があります。

## 対象読者と前提

- 13章でブラウザ版Claudeを一通り使った人
- Appendix「ワークフローツール」で、生成AIを1つのノードとして組み込む構図を確認した人
- コードは書かないが、PC上のファイル整理や下書き作成までAIに任せてみたい人

ターミナル（コマンドを入力する画面）を日常的に使っている必要はありません。本付録の手順では、必要な場面でだけターミナルを開きます。

Claude Codeの位置づけは、Claudeの製品群全体の中で確認しておくと整理しやすくなります。Cowork・Computer Use・Claude Codeの3製品とブラウザ版の比較は、[13章「Claudeの主な4製品」節](13-claude.md)の表と図にまとめてあります。あらかじめそちらへ目を通しておくと、本付録で扱う範囲が把握しやすくなります。

## Claude Codeは自分のPC上で動くClaude

**Claude Code**は、Anthropic社が提供するClaude公式のクライアントです。ブラウザ版のClaudeがAnthropic側のサーバ上で会話を完結させるのに対し、Claude Codeは自分のPC上で動き、許可した範囲のローカルファイルを読み書きできます。

Claude Codeにはいくつかの利用形態が用意されており、いずれも同じClaude Code本体を呼び出します。

| 利用形態 | 主な使いどころ |
| ---- | ---- |
| CLI（ターミナル） | ターミナル上でファイルとコマンドを直接の対象にする |
| デスクトップアプリ（Mac／Windows） | グラフィカル画面からClaude Codeの機能を呼び出す |
| Webアプリ | ブラウザ内で同等の対話を行う。ローカルファイルは扱わない |
| IDE拡張（VS Code／JetBrains） | エディタ内で呼び出し、編集中のファイルを対象にする |

非エンジニアの場合は、デスクトップアプリかIDE拡張から始めるのが穏当です。CLI版は、ターミナル操作に抵抗がなく、キーボード主体で作業を進めたい人に向きます。

## ブラウザ版との違いはローカルファイルとコマンド実行の有無

ブラウザ版Claudeとの違いを表にまとめます。

| 観点 | ブラウザ版Claude | Claude Code |
| ---- | ---- | ---- |
| 動作環境 | Anthropic側のサーバ | 自分のPC（CLI／IDE／デスクトップアプリ） |
| ファイル操作 | アップロードしたファイルのみ | 許可した範囲のローカルファイルを読み書き |
| コマンド実行 | 原則なし | 許可した範囲でシェルコマンドを実行 |
| 向く仕事 | 会話、資料作成、1ファイル単位の編集 | フォルダ全体の変更、スクリプト作成、連続作業 |

呼び出しているモデルはどちらも同じClaudeで、回答の質そのものに差はありません。違いは利用者のPCに対してどこまで操作を許すかにあり、Claude Codeはローカルファイルの読み書きとコマンド実行までを担います。

## インストールから最初の対話まで

最短ルートは、公式ドキュメントの手順に従ってCLIかデスクトップアプリを導入し、空のフォルダで会話を始めることです。インストール手順は公式ドキュメントを一次ソースにしてください。執筆時点で、Node.jsが入っているPCの最短コマンドは次の1行です。

```bash
npm install -g @anthropic-ai/claude-code
```

起動後、対象フォルダを指定してClaudeに依頼を送ると、そのフォルダ内のファイルを読み込みます。たとえば、議事録ファイルを並べたフォルダで次のように依頼すると、要約の下書きが返ってきます。

```text
このフォルダの議事録を読んで、先月の意思決定を箇条書きで要約してください。
出典としてファイル名も併記してください。
```

最初に確認しておきたいのは、Claude Codeが実際に読み書きするファイルと、実行を試みるコマンドの内容を、画面上で逐一確認できる点です。許可と拒否の判断は利用者側に残されており、利用者の確認なく作業を進める形ではありません。

## 先に把握しておく4つの用語

Claude Codeを少し深く使う場面では、次の4語を頭に置いておくと混乱しにくくなります。詳しく覚える必要はなく、辞書のように後から参照できれば十分です。

| 用語 | 一言で | 初見でのおすすめ度 |
| ---- | ---- | ---- |
| スラッシュコマンド | `/help` のように先頭に `/` を付けて呼ぶ定型処理 | 高（まず `/help` を打つ） |
| サブエージェント | 特定の役割に絞った別のClaudeを呼び出す機能 | 中（慣れてから） |
| MCPサーバ | 外部サービスとClaude Codeをつなぐ共通の差込口 | 中（連携先ができてから） |
| フック | 特定のタイミングで自動的に動く処理 | 低（手になじむまでは必要ない） |

### スラッシュコマンド

入力欄の冒頭で `/` を押すと、利用できるコマンドの一覧が出ます。最初に試しておきたいのは `/help` で、迷ったときの戻り場所として使えます。

### サブエージェント

ひとつの会話の中で、「コードを読む役」「テストを実行する役」のように役割を分けた別のClaudeを呼び出せる仕組みです。[7章で扱った「エージェント」](07-terminology.md)の考え方を、Claude Code側で形にしたものです。

### MCPサーバ

MCPはModel Context Protocolの略で、[4章で扱ったツール呼び出し](04-external-system-integration.md)を複数のクライアントから共通の作法で扱えるよう規格化したものです。連携先ごとにMCPサーバを用意しておくと、Claude Codeから同じ手順で呼び出せます。

### フック

「コマンド実行の直前にリントを走らせる」「会話終了時に通知を出す」など、特定のタイミングで自動的に動く処理を差し込む仕組みです。便利ですが、設定の誤りが意図しない動作につながることもあるため、はじめのうちは設定を加えずに運用できます。

## 非エンジニアでも実用になる場面

開発者以外の読者がClaude Codeに頼みやすいのは、コードよりもむしろPC上の定型仕事です。たとえば次のような作業が射程に入ります。

- 1つのフォルダに集めた資料（議事録、ヒアリングメモ、CSV）の要約と横断比較
- スクリーンショットや写真のリネームと、日付別のフォルダ分け
- 調査の下書き（章立て付きのMarkdownで出力し、自分で清書する進め方）
- 簡単なスクリプトの生成と動作確認（「このCSVから重複を除いたリストを出して」など）
- Google WorkspaceやSlackとの連携（MCP経由でコネクタを追加してから）

逆に、Claude Codeが向かない領域もあります。デザインの最終仕上げ、長尺動画の編集、推敲を重ねるライティングなどは、ブラウザ版Claudeや専用ツールのほうが向きます。仕事に応じて使い分けるのが順当です。

## 始める前に押さえておく5項目

CLIやMCPといった用語が並びますが、Claude Codeは自分のPCに対して書き込みやコマンド実行を担う道具です。次の5項目を最初に押さえておくと、意図しない操作の影響範囲を小さく保ちやすくなります。

- 許可ダイアログを読み飛ばさない — ファイルの書き込みやコマンド実行の許可ダイアログには、毎回目を通してから承認する
- 作業用フォルダを分ける — 本番のデータが入った場所で試運転をせず、一時的な作業用フォルダを用意する
- Gitなどの履歴の下に置く — 生成結果は履歴に残し、あとから差し戻せる状態を保つ
- 個人情報・顧客情報の扱いを事前に確認する — 機微なデータを扱う前に、[9章「個人利用編」](09-security-individual.md)と[10章「エージェント時代のガバナンス」](10-security-agent-era.md)を参照する
- 自動化は段階的に広げる — フックや自走エージェントを最初から組むと、意図しない操作の影響範囲が大きくなる

要点は1つに集約できます。小さなフォルダで、人の確認を経由しながら、徐々に範囲を広げる運用です。この運用を保つあいだは、想定外のファイル変更やコマンド実行に至りにくくなります。

## ブラウザ版・IDE拡張との使い分け

同じClaudeの製品群の中でも、動作環境と権限範囲が違うため、向く仕事は変わります。状況別の選択は次のとおりです。

| 状況 | 第一選択 | 補足 |
| ---- | ---- | ---- |
| 会話しながら1ファイルを仕上げたい | ブラウザ版Claude | Artifactsで成果物を整理する |
| 自分のPC上のファイルをまとめて扱いたい | Claude Codeデスクトップ／CLI | フォルダ単位で許可する |
| エディタ内で編集しながら使いたい | IDE拡張（VS Code／JetBrains） | エディタ側の差分表示をそのまま使える |
| PCが手元になく、ブラウザだけ使える | ブラウザ版Claude／Webアプリ | ファイル操作は対象外。資料生成に絞る |

判断軸は「どれが優れているか」ではなく、「いま手元の作業の規模に見合うか」です。仕上げたいファイルが1つのときと、フォルダ横断の連続作業のときとでは、向く形態が変わります。

## まとめ

- Claude Codeは自分のPC上で動くClaudeであり、ブラウザ版との差はローカルファイルの読み書きとコマンド実行を担う点
- 利用形態はCLI・デスクトップアプリ・Webアプリ・IDE拡張の4つで、呼び出すClaude Code本体は共通
- 最初は小さなフォルダで、許可ダイアログを毎回確認しながら進めると、意図しない操作の影響を抑えやすい
- フックやMCPサーバは、普段使いの形ができてから足すのが順当
