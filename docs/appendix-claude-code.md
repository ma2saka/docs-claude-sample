# Appendix: Claude Code

Claude Codeは、Claudeの製品群のうち利用者のPC上で動く製品の1つで、許可した範囲のローカルファイルの読み書きとコマンド実行までを担います。本付録は、ブラウザ版のClaudeから一歩進んでClaude Codeを利用するときの位置づけ、チャット版との違い、最初の運用方針を扱います。

開発寄りに作られた道具ですが、利用者のPC上の資料整理や定型作業の自動化など、非エンジニアの仕事にも当てはまる利用範囲があります。本付録の後半は、その範囲を中心に扱います。

## 対象読者と前提

- [13章](13-claude.md)でブラウザ版Claudeを一通り利用した人
- [Appendix: ワークフローツール](appendix-workflow-tools.md)で、生成AIが処理の1ステップとして動く構成に目を通した人
- 「コードは書かないが、PC上のファイルや定型作業までAIに任せてみたい」立場の人

コマンドラインを日常的に利用する必要はありません。必要に応じてターミナル（コマンドを打ち込む画面）を開く場面もある、くらいの温度感でお読みください。

Claudeの4製品の役割分担は、[13章「Claudeの主な4製品はブラウザ版とPC上で動く3つ」](13-claude.md)の表と図に整理されています。本付録は、そのうちClaude Codeを掘り下げる位置づけです。13章の該当節の表と図は、本付録で踏み込む話のスケール感を見定める基準として参照できます。

## Claude Codeは公式CLIで、4つの呼び出し方がある

**Claude Code**は、Anthropic社が提供する公式のコマンドラインツールです。ブラウザ版Claudeが`https://claude.ai`上で動作するのに対し、Claude Codeは利用者のPC上で動作し、許可した範囲のローカルファイルを読み書きします。

中身は同じClaude Codeで、起動の入口が4つ用意されています。

| 入口 | 利用感の概要 |
| ---- | ---- |
| CLI（ターミナル） | 本家。手元のファイルとコマンドを直接対象にする |
| デスクトップアプリ（Mac／Windows） | GUIからClaude Codeの機能を呼び出す |
| Webアプリ | ブラウザ内でほぼ同等の操作。ローカルのファイルは扱わない |
| IDE拡張（VS Code／JetBrains） | エディタの中でそのまま呼び出す |

非エンジニアの利用者は、まずデスクトップアプリかエディタ拡張から始めると、ターミナル操作の負担なくClaude Codeの動きを把握できます。CLI版は、ターミナルでの操作に抵抗がなく、キーボード主体で作業を進めたい人に向きます。

## チャット版との違いは実行場所と権限範囲にある

ブラウザ版のClaudeとClaude Codeの差は、回答の品質ではなく、動作環境と利用者のPCに対する権限の範囲です。両者の対比は次のとおりです。

| 観点 | ブラウザ版Claude | Claude Code |
| ---- | ---- | ---- |
| 実行場所 | Anthropic側のサーバ | 利用者のPC（またはIDE／デスクトップアプリ） |
| ファイルの扱い | アップロードしたものだけ | 許可した範囲のローカルファイルを読み書き |
| コマンド実行 | 原則なし | 許可した範囲でシェルコマンドを実行 |
| 向く作業 | 会話、資料作成、1ファイルの編集 | フォルダ全体の変更、スクリプト作成、連続作業 |

呼び出すモデルは同じで、Claude Codeはこれに加えてローカルのファイル読み書きとコマンド実行を担います。利用者のPC上で一連の作業を連続して進めたい場面が、Claude Codeを選ぶ判断軸に当たります。

## 最初の試用は空フォルダで会話を始める

もっとも手軽な始め方は、公式ガイドに沿ってCLIかデスクトップアプリを入れ、空のフォルダで会話を開始することです。インストール手順は公式ドキュメントを一次ソースにしてください。執筆時点での最短ルートは、Node.jsが入っているPCで次の1行です。

```bash
npm install -g @anthropic-ai/claude-code
```

起動後、フォルダを指定してClaudeに話しかけると、そのフォルダの中のファイルをClaude Codeが参照します。たとえば、議事録ファイルを並べたフォルダで次のように依頼すると、それらしい下書きが返ってきます。

```text
このフォルダの議事録を読んで、先月の意思決定を箇条書きで要約してください。
出典としてファイル名も併記してください。
```

ここで確認しておきたいのは、Claude Codeが実際にどのファイルを参照し、どのコマンドを実行しようとしているかが画面上に逐一表示される点です。利用者は表示された内容に対して許可／拒否を選びます。Claude Code側が利用者の同意を経ずに操作を進める作りではありません。

## Claude Codeに固有の4用語は名前だけ押さえておけばよい

Claude Codeを少し深く利用したくなった場面で目にする用語のうち、最初に意識合わせしておきたいものが4つあります。深く理解する必要はなく、名前と役割が頭にあれば、ドキュメントや画面の指示で迷いません。

| 用語 | 一言で | 初見でのおすすめ度 |
| ---- | ---- | ---- |
| スラッシュコマンド | `/help` のように先頭に `/` を付けて呼ぶ定型処理 | 高（まず `/help` を打つ） |
| サブエージェント | 役割を限定したClaude Codeを呼び出す仕組み | 中（慣れてから） |
| MCPサーバ | 外部サービスとClaude Codeをつなぐ共通の差込口 | 中（連携先ができてから） |
| フック | 特定のタイミングで自動的に動く処理 | 低（手元の操作になじむまでは利用しない） |

### スラッシュコマンド

画面内で `/` を入力すると、利用できるコマンドの一覧が表示されます。最初に押さえておきたいのは `/help` です。迷ったら `/help` に戻る、くらいの距離感で利用すれば差し支えありません。

### サブエージェント

ひとつの会話の中で「コードを読む担当」「テストを実行する担当」のように、役割を限定したClaude Codeを呼び出せる仕組みです。[7章](07-terminology.md)で扱ったエージェントの考え方を、Claude Code側で構成したものに当たります。

### MCPサーバ

MCPはModel Context Protocolの略で、[4章](04-external-system-integration.md)で扱ったツール呼び出しを共通化したプロトコルです。連携先ごとにMCPサーバを用意しておくと、Claude Codeから同じ作法でアクセスできます。

### フック

「コマンドを実行する直前にリントを実行する」「会話が終わったら通知を出す」のように、特定のタイミングで自動的に動く処理を差し込む仕組みです。便利ですが、設定ミスが意図しない動作につながりやすいため、利用は手元の操作になじんでからにします。

## 非エンジニアの実用は資料整理と定型作業の自動化に集中する

「自分は開発者ではない」という立場の利用者にとって、Claude Codeの利用価値は、コードを書くこと自体よりも、PC上の定型作業をまとめて頼める点に現れます。具体例として、次のような仕事が射程に入ります。

- ひとつのフォルダに集めた資料（議事録、ヒアリングメモ、CSV）の要約と横断比較
- スクリーンショットや写真のリネームと、日付別のフォルダ分け
- 調査の下書き（章立て付きのMarkdownで出力し、利用者が清書する進め方）
- 簡単なスクリプトの生成と動作確認（「このCSVから重複を除いたリストを出して」など）
- Google WorkspaceやSlackへの連携（MCP経由でコネクタを足してから）

一方、Claude Codeに向かない作業もあります。デザインの最終仕上げ、長尺の動画編集、文体の細かな推敲などは、ブラウザ版Claudeや専用ツールのほうが向きます。

## 最初の運用方針は「小さく分けて、人の確認を経由する」

CLIやMCPといった語感が前に立ちますが、Claude Codeは利用者のPCに対してファイルの読み書きとコマンドの実行を担う道具です。次の点を最初に意識合わせしておくと、意図しない操作の影響範囲を狭く保てます。

- 許可ダイアログを読み飛ばさない — ファイルの書き込みやコマンド実行の許可ダイアログには、目を通してから承認する
- 対象フォルダを分ける — 本番のデータが入った場所では試さず、一時的な作業用フォルダを用意する
- Gitなどの履歴の下に置く — 生成結果は履歴を残し、あとから差し戻せる状態を保つ
- 個人情報・顧客情報の扱いは別途確認する — [9章](09-security-individual.md)と[10章](10-security-agent-era.md)に目を通したうえで、機微なデータに利用する
- 自動化は段階的に広げる — 最初からフックや自走エージェントを組むと、設定ミスから意図しない動作が起きた場合の影響範囲が大きくなる

要点を1行にまとめると「小さなフォルダで、人の確認を経由しながら、徐々に広げる」になります。この方針を外さないうちは、Claude Codeの利用範囲は意図したものから外れにくくなります。

## ブラウザ版・IDE拡張との使い分けは作業の規模で決める

同じClaudeブランドの中でも、動作環境と扱える範囲が違うため、向く作業も変わります。仕事に応じた使い分けの目安は次のとおりです。

| 状況 | 第一選択 | 補足 |
| ---- | ---- | ---- |
| 会話しながら1ファイルを仕上げたい | ブラウザ版Claude | Artifactsで成果物を整理する |
| 自分のPCのファイルをまとめて操作したい | Claude Codeデスクトップ／CLI | フォルダ単位で許可する |
| コードをエディタ内で編集したい | IDE拡張（VS Code／JetBrains） | エディタ側の差分表示をそのまま利用できる |
| 外出先でPCが手元にない | ブラウザ版Claude／Webアプリ | ファイル操作は諦め、資料生成に集中 |

選択の軸は「どれが優れているか」ではなく、「いま向き合っている作業の規模に合うか」です。仕上げたいファイルが1つなのか、フォルダ横断の連続作業なのかで、向く呼び出し方は変わります。

## まとめ

- Claude CodeはAnthropic公式のCLIで、ブラウザ版との差はローカルファイルの読み書きとコマンド実行ができるかにある
- 起動の入口はCLI・デスクトップアプリ・Webアプリ・IDE拡張の4つで、中身は同じ
- 最初は小さなフォルダで試し、許可ダイアログを目視で確認するだけで、意図しない操作の多くは避けられる
- 非エンジニアでも、資料整理・スクリプト生成・SaaS連携に実用的な範囲がある
